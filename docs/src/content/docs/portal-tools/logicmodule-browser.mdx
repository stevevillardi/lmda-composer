The LogicModule Browser lets you explore all LogicModules in your portal and import their scripts into LMDA Composer for editing and testing.

## Opening the Browser

<CardGrid>
  <FeatureCard title="Welcome Screen" icon="rocket" color="orange">
    Click **Browse Modules** or the modules icon on the welcome screen.
  </FeatureCard>
  <FeatureCard title="Command Palette" icon="search" color="purple">
    Press <kbd>Cmd</kbd>+<kbd>Shift</kbd>+<kbd>P</kbd> and type "Browse" to open quickly.
  </FeatureCard>
  <FeatureCard title="Actions Menu" icon="list" color="blue">
    Click **Actions → Browse LogicModules** in the toolbar.
  </FeatureCard>
</CardGrid>

Type in the search box to filter modules by:
- Module name
- Display name  
- Description

Search is case-insensitive and matches partial strings.

- Module display name
- Module type badge

- Scroll to load more results (first 100 shown initially)

## Loading a Script

<Steps>

1. **Select a module** from the list on the left

2. **Choose the script type** you want to load (AD, Collection, etc.)

3. **Click Load Script** to open it in a new editor tab

</Steps>

The script opens with:
- **Module binding** – linked to the portal module for committing changes
- **Correct language** – Groovy or PowerShell automatically selected
- **Appropriate mode** – Execution mode matches the script type

<Aside type="note" title="Unsaved Changes">
  If you have unsaved changes in the current tab, a confirmation dialog appears. Choose to save, discard, or cancel.
</Aside>

### Creating a Copy

To work on a script without affecting the original:

1. Load the script normally
2. Click **Actions → Save As** to save locally
3. The local copy is independent of the module

## Module Binding

When you load a module script, the tab becomes "bound" to that module:

<Tabs>
  <TabItem label="Bound Tab" icon="external">
    When loaded from a LogicModule:

    - Shows a link icon <Icon name="external" size="1rem" /> on the tab
    - **Commit** button is available to push changes
    - Module name displays in the tab
    - Portal binding is remembered
    - Can edit module details (metadata)
  </TabItem>
  
  <TabItem label="Unbound Tab" icon="document">
    Local files or new scripts:

    - No link icon on the tab
    - No Commit option (must bind first)
    - Generic tab name
    - Not associated with any portal
    - Cannot edit module details
  </TabItem>
</Tabs>

## Import Options

### Import from LogicModule Exchange (LMX)

Access community modules:

<Steps>

1. Click **Import from LMX** on the welcome screen

2. Browse or search the LogicModule Exchange catalog

3. Import the module into your portal

4. Then browse and load the script using the LogicModule Browser

</Steps>

### Direct URL Import

You can open modules directly from LogicMonitor:
- From the **Manage** dropdown on a module → **Open in LMDA Composer**
- From the **LM Exchange** when viewing a module → **Open in LMDA Composer**
- LMDA Composer parses the link and loads the module automatically

## Pagination

For portals with many modules:

- Initial load shows the first **100 modules**
- Scroll to the bottom to load more
- "Loading more..." indicator appears during fetch
- Search filters are applied server-side for efficiency

## Troubleshooting

<Aside type="caution" title="No Modules Displayed">
  1. Verify you're logged into a LogicMonitor portal
  2. Check that a portal is selected in the Context dropdown
  3. Try selecting different module type filters
  4. Refresh the browser with <kbd>F5</kbd>
</Aside>

<Aside type="caution" title="Can't Find a Module">
  1. Clear all search filters
  2. Ensure all module types are enabled
  3. Verify the module exists in **Settings → LogicModules**
  4. The module may be in a different portal
</Aside>

<Aside type="caution" title="Script Preview Empty">
  Some modules don't have traditional scripts:
  - SNMP-based modules may only have OID configurations
  - Some modules use DataSource Templates
  - Check if the script is in a different section (AD vs Collection)
</Aside>

## Best Practices

<CardGrid>
  <FeatureCard title="Organize Your Work" icon="book" color="blue">
    Use descriptive tab names when loading multiple scripts. Save local copies before making major changes.
  </FeatureCard>
  <FeatureCard title="Test Module Scripts" icon="play" color="green">

  </FeatureCard>
  <FeatureCard title="Making Changes" icon="edit" color="purple">
    Load → Edit → Test with Run Script → Verify parsed output → Commit changes back to the portal.
  </FeatureCard>
