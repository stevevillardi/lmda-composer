## History Entry Details

Each execution entry shows:

| Field | Description |
|-------|-------------|
| Status | <StatusBadge status="success" label="Success" /> or <StatusBadge status="error" label="Error" /> |
| Name | Tab or module name |
| Timestamp | When it was executed |
| Duration | How long it took |
| Mode | Execution mode used |
| Language | Groovy or PowerShell |
| Collector | Which collector ran it |
| Device | Target device (if set) |
| Output Preview | First few lines of output |

## Reloading Scripts

### Load to Editor

Click **Load** on a history entry to:

<Steps>

1. Open a new tab

2. Populate with the historical script

3. Set language and mode automatically

4. Ready for editing and re-running

</Steps>

### Portal Mismatch Warning

If the history entry is from a different portal:

<Aside type="caution" title="Portal Mismatch">
  A warning dialog appears with options:
  - **Switch Portal**: Change to the original portal
  - **Load Anyway**: Load as freeform script
  - **Cancel**: Abort the load
</Aside>

### Module Binding

Historical scripts from modules:
- Can restore module binding if portal matches
- Falls back to freeform if unavailable

## History Management

### History Limit

Configure the maximum history entries:

<Steps>

1. Open Settings (<kbd>Ctrl</kbd>+<kbd>,</kbd>)

2. Find "History Size"

3. Set desired limit (10-500, default is **50**)

4. Oldest entries removed when limit exceeded

</Steps>

### Clearing History

To clear all history:

1. Click **Clear** button in panel
2. Confirm the action
3. All entries removed permanently

<Aside type="note">
  Individual entries cannot be deleted separately. Use the clear function or let them naturally expire based on the history limit.
</Aside>

## What's Stored

| Data | Stored? |
|------|---------|
| Script content | ✅ Yes |
| Output | ✅ Yes |
| Collector info | ✅ Yes |
| Device info | ✅ Yes |
| Timestamp | ✅ Yes |
| Duration | ✅ Yes |
| Tab/module name | ✅ Yes |
| Portal binding | ✅ Yes |

### Storage Location

History is stored:
- In browser local storage
- Per-browser (not synced)
- Persists across sessions
- Subject to size limits

<Aside type="note" title="Storage Limits">
  Large outputs or many entries may impact browser storage. Very large histories may trigger automatic cleanup.
</Aside>

## Privacy Considerations

History may contain:
- Device information
- Property values (from token substitution)
- API responses (if printed)
- Sensitive configuration data

Clear history when:
- Switching between projects
- Working on sensitive systems
- Sharing the browser/computer
