This guide walks you through connecting to your LogicMonitor portal and running your first script.

## Getting Started

<Steps>

1. ### Log into LogicMonitor

   Open a browser tab and log into your LogicMonitor portal (e.g., `https://yourcompany.logicmonitor.com`).

   <Aside type="note">
     LMDA Composer uses your existing browser session—no API credentials needed. The extension automatically detects your active sessions.
   </Aside>

2. ### Open LMDA Composer

   Click the LMDA Composer icon in your browser toolbar. The composer opens in a new tab displaying the welcome screen.

   <img src="/lmda-composer/getting-started/quickstart-welcome.png" alt="The welcome screen after opening LMDA Composer" style="border-radius: 0.5rem; margin: 1rem 0;" />

3. ### Select Your Context

   Before running a script, configure your execution context using the toolbar dropdowns:

   - **Portal** — Select your LogicMonitor portal from detected sessions (green indicator = connected)
   - **Collector** — Choose which collector runs your scripts (only online collectors can execute)
   - **Device** — Optional. Select a device if your script needs host properties like `system.hostname`

   <img src="/lmda-composer/getting-started/quickstart-context.png" alt="The context dropdown showing Portal, Collector, and Device selection" style="border-radius: 0.5rem; margin: 1rem 0;" />

4. ### Write Your First Script

   Click **New Script** on the welcome screen, or press <kbd>Ctrl</kbd>+<kbd>K</kbd>, <kbd>N</kbd> / <kbd>⌘</kbd>+<kbd>K</kbd>, <kbd>N</kbd> to create a new script tab.

   Try this simple script:

   ```groovy
   println "Hello from LMDA Composer!"
   println "Current time: " + new Date()
   return 0
   ```

5. ### Run the Script

   Click the **Run Script** button or press <kbd>Ctrl</kbd>+<kbd>Enter</kbd> / <kbd>⌘</kbd>+<kbd>Enter</kbd>.

   Watch the output appear in the Output Panel below the editor. A green "Complete" badge indicates successful execution.

   <img src="/lmda-composer/getting-started/quickstart-output.png" alt="Output panel showing successful execution with Complete status" style="border-radius: 0.5rem; margin: 1rem 0;" />

</Steps>

## Understanding the Output

The Output Panel provides three views depending on your execution mode:

<Tabs>
  <TabItem label="Raw Output" icon="document">
    The complete text output from your script, exactly as returned by the collector.
  </TabItem>
  <TabItem label="Parsed" icon="list-format">
    Structured view showing:
    - **AD Mode** — Discovered instances in a table
    - **Collection Mode** — Datapoints with values
    - **Batch Mode** — Grouped datapoints by instance
  </TabItem>
  <TabItem label="Validation" icon="approve-check-circle">
    Errors and warnings based on the current mode's output format requirements. Green checkmark means all output is valid.
  </TabItem>
</Tabs>

## Alternative: Create a LogicModule

Instead of a standalone script, you can create a complete LogicModule using the guided wizard:

1. Click **New LogicModule** on the welcome screen (requires portal connection)
2. Choose your module type (DataSource, ConfigSource, PropertySource, etc.)
3. Configure basic info, script settings, and requirements
4. The wizard creates a properly structured module ready for development

<Aside type="tip" title="Seven Module Types">
  The wizard supports DataSource, ConfigSource, TopologySource, PropertySource, LogSource, EventSource, and DiagnosticSource modules.
</Aside>

## Pro Tips

<Aside type="tip" title="Use the Command Palette">
  Press <kbd>Ctrl</kbd>+<kbd>Shift</kbd>+<kbd>P</kbd> / <kbd>⌘</kbd>+<kbd>Shift</kbd>+<kbd>P</kbd> to open the command palette. It provides quick access to all features and shows keyboard shortcuts.
</Aside>

<Aside type="tip" title="Launch from LogicMonitor">
  When viewing a module in LogicMonitor, use the **Manage** dropdown and select **Open in LMDA Composer** to open the module directly. You can also open modules from the LM Exchange.
</Aside>
