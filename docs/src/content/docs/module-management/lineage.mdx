Module Lineage displays the release history of LogicMonitor's core modules, showing previous versions published to the Exchange.

## What Is Module Lineage?

<Aside type="caution" title="Important Distinction">
  Module Lineage shows **LogicMonitor's published module versions** from the Exchangeâ€”not a history of changes you've made to modules. There is currently no way to view previous versions of user modifications.
</Aside>

Lineage provides visibility into how LogicMonitor has updated and improved their core modules over time:

<CardGrid>
  <FeatureCard title="Exchange History" icon="history" color="blue">
    View all versions LogicMonitor has published for a core module.
  </FeatureCard>
  <FeatureCard title="Release Notes" icon="file-text" color="green">
    See what LogicMonitor changed in each release.
  </FeatureCard>
  <FeatureCard title="Version Comparison" icon="diff" color="purple">
    Compare different Exchange releases side-by-side.
  </FeatureCard>
  <FeatureCard title="Update Tracking" icon="refresh-cw" color="orange">
    Identify what's new in the latest Exchange version.
  </FeatureCard>
</CardGrid>

---

## When Is Lineage Available?

Lineage is only available for **LogicMonitor core modules** that have been published to the Exchange. This includes:

- DataSources from the LogicMonitor Exchange
- ConfigSources, PropertySources, EventSources, and TopologySources from LM
- Any module with an Exchange-published history

<Aside type="note" title="Not Available For">
  - Custom modules you've created from scratch
  - Modules without Exchange history
  - Local files (not bound to a LogicModule)
</Aside>

<CardGrid>

<Aside type="note" title="Module-Bound Tabs Only">
  The Lineage button only appears for tabs loaded from a LogicModule with Exchange history.
</Aside>

## Lineage Dialog

<img src="/lmda-composer/module-management/module-lineage.png" alt="Lineage dialog showing version list and diff viewer" style="border-radius: 0.5rem; margin: 1.5rem 0;" />

### Version List

The left panel shows all Exchange versions:

| Column | Description |
|--------|-------------|
| Version | LogicMonitor's release version number |
| Date | When LogicMonitor published this version |
| Notes | Release notes or changelog summary |

Versions are sorted newest first.

### Version Details

Click a version to see:
- LogicMonitor's release notes
- Changes from the previous Exchange version
- Script and configuration updates

## Comparing Exchange Versions

### Side-by-Side Comparison

<Steps>

2. Click **Compare**

3. Choose the version to compare against

4. View the diff in the right panel

## Use Cases

### Checking for Updates

Compare your current module version against the latest Exchange release to see what LogicMonitor has improved:

<Steps>

1. Open the module in LMDA Composer

2. View Lineage

3. Compare your version against the latest Exchange version

4. Decide whether to adopt the updates

</Steps>

### Understanding Exchange Changes

When LogicMonitor releases a new version, use Lineage to understand:
- What bugs were fixed
- What new features were added
- What breaking changes might affect your customizations

### Reference Implementation

If you've customized a module and need to see the "original" version:

<Steps>

1. Open Lineage for the module

2. Find the Exchange version your module was based on

3. View the original script content

4. Compare against your customizations

</Steps>

## Limitations

<CardGrid>
  <FeatureCard title="No User History" icon="x-circle" color="red">
    Lineage does not track changes you make to modules. There's no way to view or revert to previous versions of your own edits.
  </FeatureCard>
  <FeatureCard title="Exchange Only" icon="info" color="blue">
    Only modules with Exchange history have Lineage data. Custom modules created from scratch won't have Lineage.
  </FeatureCard>
</CardGrid>

<Aside type="tip" title="Version Control for Your Changes">
  For tracking your own modifications, consider:
  - Exporting scripts to local files and using Git
  - Keeping notes of changes in module descriptions
  - Maintaining a changelog in your team's documentation
</Aside>
