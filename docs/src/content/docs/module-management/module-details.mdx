The Module Details Editor allows you to view and modify metadata for module-bound tabs. Edit properties like name, description, AppliesTo expressions, and alert configurations.

## Opening Module Details

<CardGrid>
  <FeatureCard title="Toolbar" icon="edit" color="blue">
    Click the **Module Details** button in the toolbar.
  </FeatureCard>
  <FeatureCard title="Command Palette" icon="search" color="purple">
    Press <kbd>Ctrl</kbd>+<kbd>K</kbd> and type "Module Details".
  </FeatureCard>
  <FeatureCard title="Tab Context Menu" icon="list" color="green">
    Right-click a module tab and select **Module Details**.
  </FeatureCard>
</CardGrid>

<Aside type="note" title="Module-Bound Tabs Only">
  The Module Details button only appears for tabs loaded from a LogicModule. Local files don't have module metadata.
</Aside>

## Basic Info Section

<Tabs>
  <TabItem label="Module Name" icon="tag">
    The internal identifier for the module:
    - Must be unique within the portal
    - Alphanumeric with underscores only
    - Used in API references
    - Maximum 255 characters
  </TabItem>
  <TabItem label="Display Name" icon="pencil">
    The human-readable name:
    - Shown in the LogicMonitor UI
    - Can include spaces and special characters
    - Maximum 100 characters
  </TabItem>
  <TabItem label="Description" icon="document">
    Documentation for the module:
    - Supports multi-line text
    - Explain what the module monitors
    - Include setup requirements
    - Maximum 1000 characters
  </TabItem>
  <TabItem label="Version" icon="information">
    Read-only field showing:
    - Current version number
    - Last modified date
    - Modified by user
  </TabItem>
</Tabs>

## Organization Section

### Module Group
Organize modules into groups by selecting from existing groups or creating new ones in LogicMonitor.

### Tags
Add searchable tags as comma-separated values. Examples: `custom`, `production`, `network`

### Technology Type
Categorize by technology: Windows, Linux, Network, Cloud, or Custom.

### Collector Attribute
For modules requiring specific collector capabilities: SNMP support, WMI support, or Script execution.

---

## Access Section

### AppliesTo Expression

Define which resources use this module:

```groovy
system.sysinfo =~ "Linux" && hasCategory("server")
```

Edit the expression directly or click **Test** to open the AppliesTo Toolbox.

### Group Restrictions

- Select groups from dropdown
- Multiple groups allowed
- Leave empty for no restrictions

### Collector Groups

Assign to specific collector groups for geo-specific monitoring.

## Datapoints Section

Manage collected metrics:

### Datapoint List

| Column | Description |
|--------|-------------|
| Name | Datapoint identifier |
| Type | Gauge, counter, or derive |
| Description | Documentation text |
| Alert Enabled | Whether alerting is active |

### Managing Datapoints

<Steps>

1. **Add**: Click **Add Datapoint**, enter name, select type, configure collection

2. **Edit**: Click a datapoint to modify description, post-processor, thresholds

3. **Remove**: Select and click **Remove**, then confirm

</Steps>

<Aside type="caution" title="Data Loss">
  Removing a datapoint also removes historical data for that metric. This cannot be undone.
</Aside>

- <Badge text="Error" variant="caution" /> threshold

Assign default chains or per-severity overrides.

---

## Making Changes

### Change Tracking

The editor tracks all modifications:
- Modified fields highlighted
- Section indicators show dirty state
- Undo available per field

### Staging Changes

<Steps>

1. Review all changes across sections

2. Click **Stage Changes**

3. Changes are staged for commit

4. Script changes plus metadata commit together

</Steps>

### Resetting Changes

Click **Reset Changes** to discard all modifications and revert to original values.
