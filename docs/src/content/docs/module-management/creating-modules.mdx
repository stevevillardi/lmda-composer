The Create LogicModule wizard guides you through building new modules directly from LMDA Composer. Create DataSources, ConfigSources, and other module types without leaving the editor.

## Prerequisites

| Requirement | Description |
|-------------|-------------|
| **Portal Connection** | You must be connected to a LogicMonitor portal with an active session |
| **Write Access** | Your account needs permission to create LogicModules in the portal |

<Aside type="note" title="Portal Required">
  The Create LogicModule option only appears when you have an active portal connection. If you don't see the option, ensure you have a LogicMonitor tab open in your browser.
</Aside>

---

## Starting the Wizard

Access the Create LogicModule wizard from the welcome screen:

<Steps>

1. Close all tabs or open a new LMDA Composer window to see the welcome screen

2. Click **Create LogicModule** in the Quick Start section

3. The wizard opens with module type selection

</Steps>

## Wizard Steps

### Step 1: Module Type

Select the type of LogicModule to create:

<Tabs>
  <TabItem label="DataSource" icon="seti:db">
    <ModuleTypeBadge type="datasource" />

    Collect numeric metrics from devices. Supports Active Discovery for multi-instance monitoring.
  </TabItem>
  <TabItem label="ConfigSource" icon="seti:config">
    <ModuleTypeBadge type="configsource" />

    Collect and track configuration data. Alert on config changes or validation failures.
  </TabItem>
  <TabItem label="TopologySource" icon="random">
    <ModuleTypeBadge type="topologysource" />

    Discover network topology and relationships between resources.
  </TabItem>
  <TabItem label="PropertySource" icon="tag">
    <ModuleTypeBadge type="propertysource" />

    Discover and set device properties automatically.
  </TabItem>
  <TabItem label="LogSource" icon="document">
    <ModuleTypeBadge type="logsource" />

    Collect and parse log data from devices.
  </TabItem>
  <TabItem label="EventSource" icon="bell">
    <ModuleTypeBadge type="eventsource" />

    Generate events from device data for alerting.
  </TabItem>
  <TabItem label="DiagnosticSource" icon="stethoscope">
    <ModuleTypeBadge type="diagnosticsource" />

    Run diagnostic scripts on-demand or via Diagnostic Rules.
  </TabItem>
</Tabs>

### Step 2: Basic Information

Enter the module identification:

| Field | Required | Description |
|-------|----------|-------------|
| **Name** | Yes | Technical identifier (alphanumeric, underscores, hyphens) |
| **Display Name** | DataSource/ConfigSource/EventSource only | Human-readable name shown in UI |

<Aside type="tip" title="Auto-Sync">
  The Display Name automatically syncs with the Name field until you manually edit it.
</Aside>

### Step 3: Script Configuration

Configure the scripting options:

| Option | Description |
|--------|-------------|
| **Script Language** | Choose between Groovy and PowerShell (where supported) |
| **Instance Type** | Single instance or Multi-instance with Active Discovery (DataSource/ConfigSource only) |
| **Batch Script** | Enable batch collection for multi-instance modules |
| **AD Language** | Choose language for Active Discovery script (can differ from collection) |

<Aside type="note" title="Language Availability">
  Not all module types support both languages. LogSource and EventSource only support Groovy.
</Aside>

### Step 4: Confirmation

Review your configuration before creating:

- Module type and name
- Script language selection
- Instance type and AD settings
- Default settings that will be applied

#### Initialize Local Directory

You can optionally create a local directory structure for the new module:

| Option | Description |
|--------|-------------|
| **Initialize Local Directory** | When checked, creates a folder with script files on your local disk |

This is useful if you want to:
- Use version control (Git) for your module scripts
- Have local backups of your work
- Work offline and sync later

Click **Create Module** to create the module in your portal.

## Default Settings

Each module type is created with sensible defaults:

### DataSource Defaults

| Setting | Default Value |
|---------|---------------|
| **exitCode Datapoint** | A default datapoint that captures the script exit code |
| **Collection Interval** | 5 minutes |
| **AppliesTo** | `false()` (won't match any resources until configured) |

### ConfigSource Defaults

| Setting | Default Value |
|---------|---------------|
| **RetrievalCheck** | A config check that alerts when configuration cannot be retrieved |
| **Collection Interval** | 1 hour |
| **AppliesTo** | `false()` (won't match any resources until configured) |

### LogSource Defaults

| Setting | Default Value |
|---------|---------------|
| **Resource Mapping** | Default log fields and resource mapping configuration |
| **Collection Interval** | 5 minutes |
| **AppliesTo** | `false()` (won't match any resources until configured) |

### EventSource Defaults

| Setting | Default Value |
|---------|---------------|
| **Alert Level** | Warning |
| **Effective Interval** | 60 minutes |
| **Schedule** | 30 minutes |
| **AppliesTo** | `false()` (won't match any resources until configured) |

### All Modules

<Aside type="note" title="Safe Development Mode">
  All new modules are created with `AppliesTo: false()` so they won't match any resources until you configure targeting. This prevents accidental data collection during development.
</Aside>

## After Creation

Once the module is created:

<Steps>

1. The module opens in a new tab, bound to your portal

2. Edit the collection script in the main editor

3. Use **Module Details** to configure metadata, datapoints, or config checks

4. Test your script with the **Run** button

5. **Commit** changes to save back to LogicMonitor

6. Optionally, save to a module directory for local development

</Steps>

## Troubleshooting

### Wizard Not Available

<Aside type="caution" title="Portal Connection Required">
  If you don't see the Create LogicModule option:
  - Ensure you have a LogicMonitor portal tab open
  - Check that your session is active (not logged out)
  - Refresh the LMDA Composer tab
</Aside>

### Creation Fails

Common causes of creation failures:

| Error | Solution |
|-------|----------|
| Module name already exists | Choose a unique name |
| Permission denied | Check your LogicMonitor role permissions |
| API error | Verify portal connection and try again |
